GA4 Architecture

Cross-domain attribution restored. Modeled conversions preserved through Consent Mode v2. Enhanced Conversions feeding match rates back to paid platforms. Before the tests ran.

Why It Matters

A test run on broken data isn't an experiment — it's a guess with extra steps.

Every engagement I've taken on has had at least one GA4 configuration issue that would have invalidated test results if it hadn't been caught first. Standard patterns: cross-domain handoff breaks, session stitching failures, Consent Mode misconfiguration. I fix the architecture before the program launches, not alongside it.

1. Cross-Domain Attribution

The problem pattern. A brand uses a third-party checkout system (a payments gateway, a subscription platform, a B2B quoting tool). When the buyer moves from the store domain to the checkout domain, GA4 treats it as a new session. Purchases fire — but transaction value, item data, and the original acquisition source (Google Ads, Meta, organic) all collapse into direct / none. The marketing team is making paid channel decisions on fabricated attribution.

The fix. - Enable allowLinker and verify the GA4 configuration fires consistently across both domains - Build a custom event listener that fires the GA4 purchase event directly from the third-party pixel manager using a standardized ecommerce data layer — items[], value, currency, transaction_id - Confirm in DebugView with a side-by-side session trace from ad click to order confirmation

The result. Full-funnel attribution. The brand can measure Return on Ad Spend (ROAS) accurately and A/B tests can be read all the way to the final purchase event — not just stopping at Add-to-Cart.

2. Session Stitching and Client ID Persistence

The problem pattern. User-level reporting relies on a consistent GA4 client_id across sessions. When a site has custom cookie management, a cookie consent banner that delays GA4 initialization, or a single-page-app router that doesn't consistently emit page_view events, the client_id fragments — and any user-level metric (returning visitors, LTV segments, cohort retention) becomes unreliable.

The fix. - Audit the cookie persistence — confirm _ga cookie is set with the correct domain scope (.brand.com, not www.brand.com) - Ensure the GA4 configuration tag fires before any event tags on first load, so the client_id is established before events reference it - For SPAs, configure send_page_view: false on the configuration tag and fire manual page_view events on route changes

3. Privacy-Safe Measurement

Consent Mode v2. Standardized consent signals sitewide so tagging respects user choices (GDPR / CCPA compliance) while maximizing modeled conversions for users who opt out. Configured correctly, Google's modeling fills in gaps rather than losing the signal entirely — preserving advertising targeting effectiveness even as third-party cookie access collapses.

Enhanced Conversions. Configured GTM to capture and SHA-256 hash user-provided data (email addresses, phone numbers) directly from the checkout data layer when consent is given. Passed back to Google Ads to improve match rates on conversion events. Particularly valuable for brands with longer consideration cycles where the ad click and the purchase happen days apart.

4. Custom Event Schema

Standard GA4 events (view_item, add_to_cart, begin_checkout, purchase) cover the core funnel. For testing-specific analysis I layer on custom events — variant exposure, section interaction, scroll milestone — with a consistent naming schema so every test's data can be queried identically in BigQuery or Looker Studio.

This isn't event bloat. It's the difference between reading a test at "did CVR move" and reading a test at "where in the funnel did the variant outperform."

The Operational Standard

I don't ask developers to "fix tracking." I provide the exact dataLayer schema, the trigger conditions, the QA checklist, and the DebugView verification steps. The fix is only complete when I've confirmed every payload in a live session trace — not when the developer says it's deployed.
