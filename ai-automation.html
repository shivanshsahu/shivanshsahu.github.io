The Autonomous CRO Engine

A multi-agent system where Claude orchestrates, Gemini reads live GA4 data, and specialist AI skills run the full CRO diagnostic — from detecting the biggest revenue leak to generating a prioritized experiment roadmap — without a human in the loop between steps.

Right now, a CRO engagement starts when a human opens a dashboard. The engine inverts that: it monitors GA4 continuously, detects when something meaningful shifts, decides what to audit, runs the audit using structured skills, and surfaces a treatment plan. The human's job becomes approving the work, not initiating it.

Three Agents. One System of Record.

Claude · Orchestrator
Control Plane
Receives the request or trigger. Resolves which client, which workflow, which specialist skills are needed. Loads the minimal context bundle, delegates scoped sub-tasks, reviews outputs, and writes the final state back.
Intent → routing → synthesis
Gemini · Data Plane
Google / Data Layer
Reads and writes Google-native assets: GA4 export sheets, strategy docs, client Drive folders. Each client has explicit bindings — Gemini only touches what it's pointed at, with the right credentials for that account.
Live data fetch + writeback
Codex · Execution Plane
Code Layer
Invoked only when the system itself needs code changes — patching tracking scripts, adapters, or schemas. Returns reviewable diffs. Never touches live containers without a human approval step.
Code patches → reviewable diffs

Nine Core Skills. Each Does One Thing Well.

Skills are the reasoning layer — reusable intelligence units that don't know which client they're running for. The context (page maps, funnel data, VoC, scoring rules) is injected at runtime from the client workspace.

GA4 Opportunity Engine
Reads live funnel data. Identifies the biggest leak by segment, device, and page type. Outputs a ranked scope recommendation before any audit begins.
Scope Recommendation Engine
Prevents over-auditing. Given the opportunity signal, decides which pages and which audit depth are worth running — not everything, just the right things.
Deep Audit Engine
Runs a structured audit on the chosen scope: heuristics, VoC evidence, competitor review, copy analysis, imagery critique. Outputs a diagnosis, not a list of suggestions.
Evidence Synthesizer
Connects the dots across data sources — session recordings, survey responses, review text, heatmaps — into a single coherent problem statement with confidence weighting.
Treatment Generator
Converts diagnosis into concrete experiment concepts. Each treatment includes a hypothesis, expected mechanism, success metric, and guardrails.
Test Prioritization Engine
ICE-scores the treatment backlog using agency-specific scoring rules. Outputs an ordered queue that accounts for implementation cost and current test pipeline state.
AB Analysis
Reads test results, calculates statistical significance, checks for novelty effects and segment divergence, and writes a readout that explains the psychology — not just the metric.
Learning Memory Writer
After every test, writes what was learned into the client's persistent memory — so the next test starts from a compounding base, not from scratch.
Program Manager
Tracks pipeline state, flags blockers, surfaces what's in QA vs. live vs. awaiting analysis, and keeps the Master Tracker synchronized with the actual test queue.

What a Single Request Looks Like End-to-End

Request: "Find the biggest CRO opportunity for Comforth and recommend the next experiment."

1
Resolve client context Claude
Identifies agency (Cazy Web), client (Comforth), job type (opportunity detection + experiment recommendation). Loads client manifest, page map, funnel map, backlog, and tool bindings.
2
Fetch live data Gemini
Reads the GA4 export sheet, any strategy notes doc, and designated Drive folder assets listed in the Comforth binding. Uses the right Google account credentials — no manual setup.
3
Run detection layer Skills
GA4 Opportunity Engine identifies where revenue is leaking. Scope Recommendation Engine narrows the audit to the highest-value surface — not a full-site sweep.
4
Scoped deep audit Skills
Deep Audit Engine + Evidence Synthesizer run on the chosen scope: heuristic review, VoC coding, competitor analysis, copy and imagery critique. Outputs a diagnosis, not a wishlist.
5
Generate treatment plan Skills
Treatment Generator converts the diagnosis into prioritized experiment concepts with hypotheses, success metrics, and guardrails. Prioritization Engine orders them against the live test queue.
6
Write everything back Gemini Claude
Opportunity report, audit doc, treatment plan, and backlog updates are written into the Comforth workspace. Recent-context cache is refreshed — next run starts from here, not from zero.
7
Code patching only if needed Codex
If tracking changes are required, Codex patches the canonical script and returns a reviewable diff. Never auto-deploys — the human approves code changes before they ship.
The Autonomy Vision

From "I ran the audit" to "the system found it"

The current system is semi-autonomous: a human triggers the command, and the agents do the work. The full vision removes that trigger. The GA4 Opportunity Engine runs on a schedule, detects when a page's conversion rate drops by a meaningful threshold, and files a scoped audit job without anyone asking. By the time a strategist opens their laptop in the morning, the overnight analysis is already done — with treatments ready for review.

The key design constraint that makes this possible: every run writes back state. After each job, the client workspace contains an updated recent-context cache, a refreshed memory layer, and new entries in the evidence store. The next run reads that compounded knowledge instead of re-deriving everything from raw data. The system gets smarter with each engagement, not just faster.

Now
Human-triggered, AI-executed
Strategist sends a command. Claude resolves context, runs skills, writes outputs. Human reviews and decides.
Next
Scheduled detection + triage
GA4 monitoring runs on a cron. Meaningful changes trigger automatic audit scoping and backlog suggestions. Human approves scope before deep audit runs.
Target
Fully autonomous program management
End-to-end: detect → audit → treat → prioritize → launch test → analyze → learn → repeat. Human approval gates remain for code deploys and live test launches.

Featured Build: Video Creative at Scale

Production Pipeline · Built & Deployed

Programmatic Video Generation — Bonkers Corner

Gen-Z Streetwear Brand · India · D2C

Creating platform-specific video creative for ad testing on TikTok and Meta is a production bottleneck at every D2C brand. Each SKU needs multiple variants — different hooks, different aspect ratios, different CTAs. Doing this manually with a video editor doesn't scale when you're testing 50+ products.

The solution: a Python script that reads a master product CSV, applies a standardized prompt template built and tested against brand guidelines, and submits each product to the Kling AI API for video generation. Output links are written back to a results CSV. The entire batch runs unattended.

How the Pipeline Works

01
CSV Input
Product handle, start image URL, end image URL, per-row prompt override — one row per asset
02
Preflight
HEAD/GET check on both image URLs before submitting — prevents burning API credits on broken CDN links
03
Kling API
createTask (image-to-video, start + end frame) → async threaded poll → download MP4 locally
04
Resume-safe output
Results appended to CSV as they complete. Re-run skips already-succeeded rows automatically

What "Tested Prompt" Actually Means

The default prompt in the script is the result of iterating against 20–30 outputs and locking a structure that reliably produces usable assets: "Cinematic product showcase. Smooth camera orbit left-to-right. Subtle natural body movement. Fabric reacts realistically. No warping, no glitches." Each row in the CSV can override it per-product. The pipeline logic never needs to change when the creative direction shifts.

The real unlock: The script runs submission, polling, and downloading as separate thread pools in parallel — up to 6 submissions, 10 pollers, and 4 downloaders concurrently, with rate limiting and exponential backoff. A video editor produces 3–5 assets per day. This pipeline handles a full product catalog in a single overnight run.
